# Eventos Feed Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the new `feed` microservice on port `4003` with an `eventos` module that supports event CRUD and presence confirmation.

**Architecture:** The service follows the existing NestJS + DDD structure used by `associacao` and `identidade`: domain entities and repository interfaces, application services and DTOs, infra controllers, Drizzle schemas and repositories. It owns the `athlos_feed` PostgreSQL database and is wired into Docker Compose like the other microservices.

**Tech Stack:** NestJS 11, TypeScript, Drizzle ORM, PostgreSQL 16, RabbitMQ shared module, Docker Compose.
